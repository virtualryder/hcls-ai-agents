The agent reads a real adverse-event case through the governed gateway, masks fail-closed, and cannot act on its own. Every result below is actual captured output.
Reference accelerator — not an AWS service, not AWS-supported software, not a compliance certification. openFDA is public, de-identified data (no BAA). The PHI-under-BAA variant is AWS HealthLake FHIR.

This is the answer to the two questions every reviewer asks: does the governed pattern work against a real system of record? and how do you know the output is good? The four beats below are the live demo; the scoreboard is the CI-gated quality benchmark. Nothing here is mocked.

ALLOW  3 · Governed duplicate search (real data)

decision=ALLOW  audit_id=f4fff6db  candidates=[]

PENDING_APPROVAL DENY  4 · The human authority boundary — the trust anchor

The money shot. The agent can draft, but a consequential write needs a human approval, and the irreversible ICSR submission is withheld from the agent entirely.

safety.write_case_draft (no approval) -> PENDING_APPROVAL (requires_approval=True)
safety.submit_report                  -> DENY  (PV_PROCESSOR not entitled to commit 'safety.submit_report')

Every call above was deny-by-default authorized, minted a short-lived scoped token, and wrote an append-only masked audit record. The connector can reach api.fda.gov and nothing else — proven in-cloud behind an AWS Network Firewall FQDN allow-list (a non-allowed host is dropped). See the evidence pack.
