IdP federation → token exchange → least-privilege intersection → human-authority commit (SoD) → append-only audit  — deny-by-default at every hop.

The walkthrough — 5 beats (real captured output)

1 · IdP federation — OIDC identity → verified, role-bearing claims

The platform manages no user accounts. Identity federates from the customer IdP; groups map to app roles. Verification is fail-closed (no subject → deny). RS256 is real crypto when the JWT libs are present.

federation: [real] RS256 ID token minted + verified offline (sig+aud+exp)
subject=u-pat-001  groups=['GRP-PV-PROCESSORS']  ->  role(s): ['PV_PROCESSOR']

2 · Token exchange — verified identity → short-lived, tool-scoped token

The agent never holds a standing credential. Each authorized call mints a token scoped to exactly one tool (~60s). Three abuse paths are rejected outright.

minted: jti=ed6c912b  scope=['safety.get_case']  ttl=60s  (no standing service account)
  DENY confused-deputy (token for tool A replayed at tool B): token tool scope mismatch
  DENY expired token: token expired
  DENY tampered signature: bad token signature

3 · Least-privilege intersection — the core idea

The money shot. permitted ⇔ tool ∈ agent_grant ∩ user_entitlement. The same tool, three identities: an agent can never exceed the human it acts for, and a human can't act through an agent that wasn't granted the tool.

ALLOW safety.get_case   processor via PV agent
DENY  safety.get_case   MSL user — the agent is granted it, but the human is NOT
                  ("an agent may never exceed the user's own permissions")
DENY  safety.get_case   reviewer via 07-rwe agent — agent over-reach denied

4 · Human-authority commit + separation of duties — the irreversible act

The regulated, irreversible commit (submit an ICSR) is withheld from every agent grant. It needs a bound, single-use approval where the approver is not the requestor. Self-approval and replay are both refused.

PEND safety.submit_report   human approval required
DENY self-approval refused at mint: separation of duties
ALLOW safety.submit_report   bound approval, approver commits (SoD satisfied)
DENY safety.submit_report   replay rejected: approval token already used → commit withheld

5 · Append-only audit — prove it later

Every decision is recorded, PHI-masked, with lineage and the approver identity. In production the sink is append-only (QLDB / S3 Object Lock) — the 21 CFR Part 11 audit-trail control.

6 records: ALLOW=2, DENY=2, PENDING_APPROVAL=2
  ALLOW  safety.get_case  user=u-pat-001
  DENY   safety.get_case  user=u-sam-003     (unentitled human)
  DENY   safety.get_case  user=u-mona-002    (agent over-reach)
  ALLOW  safety.submit_report  user=u-mona-002  approved_by=u-mona-002
